# 쇼핑몰 서비스 요구사항 명세서

## 1. 프로젝트 개요

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | PBShop |
| 서비스 성격 | 일반 쇼핑몰 v1 |
| 아키텍처 | REST API 기반 단일 서비스 |
| 서비스 구성 | 사용자 페이지와 분리된 관리자 페이지로 구성 |
| 인증 방식 | JWT (Access Token + Refresh Token) |

## 2. 사용자 역할 (Roles)

| 역할 | 설명 |
| --- | --- |
| GUEST | 비회원. 상품 조회, 회원가입, 로그인만 가능 |
| USER | 일반 회원. 장바구니, 주문, 결제, 배송지, 리뷰 기능 사용 가능 |
| ADMIN | 관리자. 별도 관리자 페이지에서 카테고리, 상품, 주문, 리뷰 등 운영 기능 관리 |

---

## 3. 기능 요구사항

### 3.1 회원 (Auth / User)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| AUTH-01 | 회원가입 | 이메일, 비밀번호, 이름, 전화번호를 입력해 회원가입한다. | GUEST |
| AUTH-02 | 로그인 | 이메일과 비밀번호로 로그인하고 JWT를 발급받는다. | GUEST |
| AUTH-03 | 로그아웃 | 현재 로그인 세션의 Refresh Token을 무효화한다. | USER, ADMIN |
| AUTH-04 | 토큰 갱신 | Refresh Token으로 Access Token을 재발급받는다. | USER, ADMIN |
| USER-01 | 내 정보 조회 | 로그인한 사용자의 기본 회원 정보를 조회한다. | USER, ADMIN |
| USER-02 | 내 정보 수정 | 이름, 전화번호, 비밀번호 등 기본 회원 정보를 수정한다. | USER, ADMIN |
| USER-03 | 회원 탈퇴 | 본인 계정을 탈퇴 처리한다. | USER |

### 3.2 관리자 인증 (Admin Auth)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| ADMIN-AUTH-01 | 관리자 로그인 | 관리자 전용 로그인 페이지에서 관리자 계정으로 로그인한다. | ADMIN |
| ADMIN-AUTH-02 | 관리자 로그아웃 | 관리자 세션 또는 Refresh Token을 무효화하고 로그아웃한다. | ADMIN |
| ADMIN-AUTH-03 | 관리자 토큰 갱신 | 관리자 세션 유지를 위해 Access Token을 재발급받는다. | ADMIN |
| ADMIN-AUTH-04 | 관리자 전용 접근 제어 | 관리자 페이지와 관리자 API는 관리자 계정만 접근 가능해야 한다. | ADMIN |

### 3.3 카테고리 (Category)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| CAT-01 | 카테고리 목록 조회 | 전체 카테고리 목록을 조회한다. | ALL |
| CAT-02 | 카테고리 생성 | 관리자가 관리자 페이지의 카테고리 관리 화면에서 카테고리를 생성한다. | ADMIN |
| CAT-03 | 카테고리 수정 | 관리자가 관리자 페이지의 카테고리 관리 화면에서 카테고리 이름, 순서, 상태를 수정한다. | ADMIN |
| CAT-04 | 카테고리 삭제 | 관리자가 관리자 페이지의 카테고리 관리 화면에서 더 이상 사용하지 않는 카테고리를 삭제한다. | ADMIN |

### 3.4 상품 (Product)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| PROD-01 | 상품 목록 조회 | 카테고리, 검색어, 정렬, 페이징 조건으로 상품 목록을 조회한다. | ALL |
| PROD-02 | 상품 상세 조회 | 상품 기본 정보, 옵션, 이미지, 리뷰 요약을 포함한 상세 정보를 조회한다. | ALL |
| PROD-03 | 상품 검색 | 키워드로 상품을 검색한다. | ALL |
| PROD-04 | 상품 정렬 조회 | 최신순, 낮은가격순, 높은가격순, 인기순 등으로 정렬한다. | ALL |
| PROD-05 | 상품 등록 | 관리자가 관리자 페이지의 상품 관리 화면에서 상품을 등록한다. | ADMIN |
| PROD-06 | 상품 수정 | 관리자가 관리자 페이지의 상품 관리 화면에서 상품명, 설명, 가격, 재고, 상태를 수정한다. | ADMIN |
| PROD-07 | 상품 삭제 | 관리자가 관리자 페이지의 상품 관리 화면에서 상품을 삭제하거나 비노출 처리한다. | ADMIN |
| PROD-08 | 상품 옵션 관리 | 관리자가 관리자 페이지의 상품 관리 화면에서 상품 옵션을 생성, 수정, 삭제한다. | ADMIN |
| PROD-09 | 상품 이미지 관리 | 관리자가 관리자 페이지의 상품 관리 화면에서 상품 이미지를 등록, 변경, 삭제하고 대표 이미지를 지정한다. | ADMIN |

### 3.5 장바구니 (Cart)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| CART-01 | 장바구니 조회 | 내 장바구니 목록을 조회한다. | USER |
| CART-02 | 장바구니 추가 | 상품, 옵션, 수량을 선택해 장바구니에 담는다. | USER |
| CART-03 | 장바구니 수량 변경 | 장바구니 항목의 수량을 변경한다. | USER |
| CART-04 | 장바구니 항목 삭제 | 장바구니의 특정 항목을 삭제한다. | USER |
| CART-05 | 장바구니 비우기 | 장바구니 전체 항목을 삭제한다. | USER |

### 3.6 주문 (Order)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| ORD-01 | 주문 생성 | 장바구니 또는 바로구매로 주문을 생성한다. | USER |
| ORD-02 | 내 주문 목록 조회 | 내 주문 내역을 페이징으로 조회한다. | USER |
| ORD-03 | 내 주문 상세 조회 | 주문 상품, 금액, 배송지, 결제 상태를 포함한 상세 정보를 조회한다. | USER |
| ORD-04 | 주문 취소 | 취소 가능한 상태의 주문을 취소한다. | USER |
| ORD-05 | 전체 주문 목록 조회 | 관리자가 관리자 페이지의 주문 관리 화면에서 전체 주문 목록을 조회하고 검색, 필터링한다. | ADMIN |
| ORD-06 | 주문 상태 변경 | 관리자가 관리자 페이지의 주문 관리 화면에서 주문 상태를 결제완료, 배송준비, 배송중, 배송완료 등으로 변경한다. | ADMIN |

| PAY-03 | 환불 처리 | 주문 취소 시 결제 금액을 환불 처리한다. | USER, ADMIN |

| ID | 기능 | 설명 | 권한 |

| ADMIN-02 | 주문 현황 요약 | 관리자 대시보드에서 주문 상태별 건수와 최근 주문 흐름을 조회한다. | ADMIN |

| ADMIN-05 | 카테고리 관리 화면 | 관리자 페이지에서 카테고리 목록, 검색, 생성, 수정, 삭제 기능을 제공한다. | ADMIN |
| ADMIN-06 | 상품 관리 화면 | 관리자 페이지에서 상품 목록, 검색, 필터, 상세, 등록, 수정, 상태 변경 기능을 제공한다. | ADMIN |
| ADMIN-07 | 주문 관리 화면 | 관리자 페이지에서 주문 목록, 검색, 필터, 상세, 상태 변경 기능을 제공한다. | ADMIN |
| ADMIN-08 | 리뷰 관리 화면 | 관리자 페이지에서 리뷰 목록, 검색, 상세, 삭제 기능을 제공한다. | ADMIN |

## 4. v1 제외 범위

- 판매자 계정, 판매처 관리, 판매처별 가격 관리
- 가격 비교, 가격 추이, 가격 알림
- 위시리스트, 포인트, 쿠폰, 적립금
- 상품 문의, 고객센터, FAQ, 공지사항
- 채팅, 푸시 알림, 추천, 랭킹
- 커뮤니티, 게시판, 활동 내역
- 실결제 PG 연동, 정산, 세금계산서, 반품/교환 워크플로
- 고급 운영 분석, 광고/프로모션 관리, 고급 통계 대시보드
