# 게시판 서비스 요구사항 명세서

## 1. 프로젝트 개요

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | PBPost |
| 서비스 성격 | 일반 게시판 v1 |
| 아키텍처 | REST API 기반 단일 서비스 |
| 서비스 구성 | 사용자 페이지와 분리된 관리자 페이지로 구성 |
| 인증 방식 | JWT (Access Token + Refresh Token) |

## 2. 사용자 역할 (Roles)

| 역할 | 설명 |
| --- | --- |
| GUEST | 비회원. 게시판과 게시글, 댓글 목록 조회만 가능 |
| USER | 일반 회원. 게시글 작성, 수정, 삭제, 댓글 작성, 좋아요 기능 사용 가능 |
| ADMIN | 관리자. 별도 관리자 페이지에서 게시판 관리와 게시글, 댓글 운영 조치를 수행 |

---

## 3. 기능 요구사항

### 3.1 회원 (Auth / User)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| AUTH-01 | 회원가입 | 이메일, 비밀번호, 닉네임을 입력해 회원가입한다. | GUEST |
| AUTH-02 | 로그인 | 이메일과 비밀번호로 로그인하고 JWT를 발급받는다. | GUEST |
| AUTH-03 | 로그아웃 | 현재 로그인 세션의 Refresh Token을 무효화한다. | USER, ADMIN |
| AUTH-04 | 토큰 갱신 | Refresh Token으로 Access Token을 재발급받는다. | USER, ADMIN |
| USER-01 | 내 정보 조회 | 로그인한 사용자의 기본 회원 정보를 조회한다. | USER, ADMIN |
| USER-02 | 내 정보 수정 | 닉네임, 비밀번호 등 기본 회원 정보를 수정한다. | USER, ADMIN |
| USER-03 | 회원 탈퇴 | 본인 계정을 탈퇴 처리한다. | USER |

### 3.2 관리자 인증 (Admin Auth)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| ADMIN-AUTH-01 | 관리자 로그인 | 관리자 전용 로그인 페이지에서 관리자 계정으로 로그인한다. | ADMIN |
| ADMIN-AUTH-02 | 관리자 로그아웃 | 관리자 세션 또는 Refresh Token을 무효화하고 로그아웃한다. | ADMIN |
| ADMIN-AUTH-03 | 관리자 토큰 갱신 | 관리자 세션 유지를 위해 Access Token을 재발급받는다. | ADMIN |
| ADMIN-AUTH-04 | 관리자 전용 접근 제어 | 관리자 페이지와 관리자 API는 관리자 계정만 접근 가능해야 한다. | ADMIN |

### 3.3 게시판 (Board)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| BOARD-01 | 게시판 목록 조회 | 서비스에서 제공하는 게시판 목록을 조회한다. | ALL |
| BOARD-02 | 게시판 생성 | 관리자가 관리자 페이지의 게시판 관리 화면에서 게시판을 생성한다. | ADMIN |
| BOARD-03 | 게시판 수정 | 관리자가 관리자 페이지의 게시판 관리 화면에서 게시판 이름, 설명, 노출 순서를 수정한다. | ADMIN |
| BOARD-04 | 게시판 삭제 | 관리자가 관리자 페이지의 게시판 관리 화면에서 더 이상 사용하지 않는 게시판을 삭제하거나 비활성화한다. | ADMIN |

### 3.4 게시글 (Post)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| POST-01 | 게시글 목록 조회 | 게시판별로 게시글 목록을 페이징, 검색, 정렬 조건으로 조회한다. | ALL |
| POST-02 | 게시글 상세 조회 | 게시글 본문, 작성자 정보, 좋아요 수, 댓글 수를 포함한 상세 내용을 조회한다. | ALL |
| POST-03 | 게시글 작성 | 제목과 본문을 입력해 게시글을 작성한다. | USER |
| POST-04 | 게시글 수정 | 본인이 작성한 게시글을 수정한다. | USER |
| POST-05 | 게시글 삭제 | 본인이 작성한 게시글을 삭제한다. | USER |
| POST-06 | 조회수 증가 | 게시글 상세 조회 시 조회수를 증가시킨다. | ALL |
| POST-07 | 게시글 검색 | 제목 또는 본문 키워드로 게시글을 검색한다. | ALL |
| POST-08 | 게시글 정렬 조회 | 최신순, 조회수순, 좋아요순 등으로 게시글을 정렬해 조회한다. | ALL |

### 3.5 댓글 (Comment)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| COMM-01 | 댓글 목록 조회 | 특정 게시글의 댓글 목록을 조회한다. | ALL |
| COMM-02 | 댓글 작성 | 게시글에 댓글을 작성한다. | USER |
| COMM-03 | 댓글 수정 | 본인이 작성한 댓글을 수정한다. | USER |
| COMM-04 | 댓글 삭제 | 본인이 작성한 댓글을 삭제한다. | USER |

### 3.6 좋아요 (Like)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| LIKE-01 | 게시글 좋아요 등록 | 게시글에 좋아요를 등록한다. | USER |
| LIKE-02 | 게시글 좋아요 취소 | 등록한 좋아요를 취소한다. | USER |
| LIKE-03 | 게시글 좋아요 수 조회 | 게시글의 좋아요 수를 조회한다. | ALL |

### 3.7 운영 관리 (Moderation)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| MOD-01 | 게시글 숨김 처리 | 관리자가 관리자 페이지의 게시글 관리 화면에서 운영 정책에 따라 게시글을 숨김 처리한다. | ADMIN |
| MOD-02 | 게시글 삭제 | 관리자가 관리자 페이지의 게시글 관리 화면에서 운영 정책에 따라 게시글을 삭제한다. | ADMIN |
| MOD-03 | 댓글 숨김 처리 | 관리자가 관리자 페이지의 댓글 관리 화면에서 운영 정책에 따라 댓글을 숨김 처리한다. | ADMIN |
| MOD-04 | 댓글 삭제 | 관리자가 관리자 페이지의 댓글 관리 화면에서 운영 정책에 따라 댓글을 삭제한다. | ADMIN |

### 3.8 관리자 페이지 (Admin Page)

| ID | 기능 | 설명 | 권한 |
| --- | --- | --- | --- |
| ADMIN-01 | 관리자 대시보드 조회 | 관리자 페이지 메인 화면에서 게시판, 게시글, 댓글 현황 요약을 조회한다. | ADMIN |
| ADMIN-02 | 게시판 현황 요약 | 관리자 대시보드에서 게시판별 게시글 수와 활성 상태를 조회한다. | ADMIN |
| ADMIN-03 | 게시글 현황 요약 | 관리자 대시보드에서 최근 게시글 수와 숨김, 삭제 대상 현황을 조회한다. | ADMIN |
| ADMIN-04 | 댓글 현황 요약 | 관리자 대시보드에서 최근 댓글 수와 운영 조치 대상 댓글 현황을 조회한다. | ADMIN |
| ADMIN-05 | 게시판 관리 화면 | 관리자 페이지에서 게시판 목록, 검색, 생성, 수정, 삭제 기능을 제공한다. | ADMIN |
| ADMIN-06 | 게시글 관리 화면 | 관리자 페이지에서 게시글 목록, 검색, 필터, 상세, 숨김, 삭제 기능을 제공한다. | ADMIN |
| ADMIN-07 | 댓글 관리 화면 | 관리자 페이지에서 댓글 목록, 검색, 필터, 상세, 숨김, 삭제 기능을 제공한다. | ADMIN |

## 4. v1 제외 범위

- 대댓글, 멘션, 답글 트리 구조
- 첨부파일, 이미지, 영상 업로드
- 공지사항, 상단 고정, 강조 게시글
- 신고 접수, 신고 사유 관리, 제재 이력 관리
- 알림, 북마크, 구독, 팔로우
- 실시간 채팅, 실시간 알림, WebSocket 기능
- 태그 구독, 인기글 집계, 추천 피드
- 신고 처리 대시보드, 운영자 협업 기능, 고급 통계 대시보드
